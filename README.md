## рҹӨ” Why ClawTeam?

Current AI agents are powerful вҖ” but they work in **isolation**. When facing complex tasks, you're stuck manually coordinating multiple agents, juggling context, and stitching together fragmented results.

**What if agents could think and work as a team?**

ClawTeam unlocks **Agent Swarm Intelligence** вҖ” where AI agents self-organize into collaborative teams, intelligently divide complex work, share insights in real-time, and converge on breakthrough solutions.

вҖў **рҹҡҖ Spawns specialized sub-agents** вҖ” each with dedicated environments and focus areas

вҖў **рҹ“Ӣ Designs intelligent task allocation** вҖ” with smart dependency management

вҖў **рҹ’¬ Facilitates real-time coordination** вҖ” seamless inter-agent communication

вҖў **рҹ“Ҡ Monitors team performance** вҖ” tracks progress and identifies bottlenecks

вҖў **рҹ”„ Adapts strategies dynamically** вҖ” reallocates resources and redirects efforts

#### вңЁ The Result?**
You set the vision. The swarm executes with collective intelligence.

## рҹҺҜ Swarm Intelligence in Action

<table>
<tr>
<td width="33%">

### рҹҰһ Agents Spawn Agents
The leader agent calls `clawteam spawn` to create workers. Each worker gets its own **git worktree**, **tmux window**, and **identity** вҖ” automatically.

```bash
# The leader agent runs:
clawteam spawn --team my-team \
  --agent-name worker1 \
  --task "Implement auth module"
```

</td>
<td width="33%">

### рҹӨ– Agents Talk to Agents
Workers check their inbox, update task status, and report results вҖ” all through CLI commands that are **auto-injected** into their prompt.

```bash
# A worker agent checks tasks:
clawteam task list my-team --owner me
# Then reports back:
clawteam inbox send my-team leader \
  "Auth done. All tests passing."
```

</td>
<td width="33%">

### рҹ‘Җ You Just Watch
Monitor the swarm from a tiled tmux view or a Web UI. The leader handles coordination вҖ” you intervene only when you want to.

```bash
# Watch all agents simultaneously
clawteam board attach my-team
# Or open the web dashboard
clawteam board serve --port 8080
```

## рҹҡҖ Quick Start

### вҡЎ Option 1: Let the Agent Drive (Recommended)

ClawTeam ships with a **Claude Code skill** that auto-activates. Just install and prompt:

```
"Build a web app. Use clawteam to split the work across multiple agents."
```

The agent will automatically create a team, spawn workers, assign tasks, and coordinate вҖ” using `clawteam` CLI commands under the hood.

### рҹ”§ Option 2: Drive It Manually

```bash
# 1. Create a team (you become the leader)
clawteam team spawn-team my-team -d "Build the auth module" -n leader

# 2. Spawn worker agents вҖ” each gets a git worktree, tmux window, and identity
clawteam spawn --team my-team --agent-name alice --task "Implement the OAuth2 flow"
clawteam spawn --team my-team --agent-name bob   --task "Write unit tests for auth"

# 3. Workers auto-receive a coordination prompt that teaches them to:
#    вң… Check tasks:    clawteam task list my-team --owner alice
#    вң… Update status:  clawteam task update my-team <id> --status completed
#    вң… Message leader: clawteam inbox send my-team leader "Done!"
#    вң… Report idle:    clawteam lifecycle idle my-team

# 4. Watch them work side-by-side
clawteam board attach my-team
```

## рҹӨ– How Agents Use ClawTeam

When an agent is spawned via `clawteam spawn`, it receives an **auto-injected coordination prompt**:

```
## Coordination Protocol (auto-injected into every spawned agent)

- рҹ“Ӣ Check your tasks: clawteam task list <team> --owner <your-name>
- в–¶пёҸ Start a task:     clawteam task update <team> <id> --status in_progress
- вң… Finish a task:    clawteam task update <team> <id> --status completed
- рҹ’¬ Message leader:   clawteam inbox send <team> leader "status update..."
- рҹ’¬ Message teammate: clawteam inbox send <team> <name> "info..."
- рҹ“Ё Check inbox:      clawteam inbox receive <team>
- рҹҳҙ Report idle:      clawteam lifecycle idle <team>
```

This means **any CLI agent** can participate in a ClawTeam team вҖ” it just needs to run shell commands. No custom SDK, no API integration, no framework lock-in.

<summary><h3>рҹ”§ Core Commands</h3></summary>

```bash

clawteam task list <team> --status blocked --owner worker1
clawteam task wait <team> --timeout 300

# рҹ’¬ Messaging
clawteam inbox send <team> <to> "message"
clawteam inbox broadcast <team> "message"
clawteam inbox receive <team>             # consume messages
clawteam inbox peek <team>                # read without consuming

# рҹ“Ҡ Monitoring
clawteam board show <team>                # terminal kanban
clawteam board live <team> --interval 3   # auto-refresh
clawteam board attach <team>              # tiled tmux view

```

| Command | `claude` | `clawteam spawn tmux codex ...` |
| Workspace | `auto` (git worktree) | `--no-workspace` |
| Permissions | skip | `--no-skip-permissions` |

| Transport | How it works | When to use |
|-----------|-------------|-------------|
| **file** (default) | JSON files in inbox dirs | Single machine, shared FS |
| **p2p** | ZeroMQ PUSH/PULL + file fallback | Low-latency, auto-fallback |
